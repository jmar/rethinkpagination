Nowadays Pagination has three major problems

Too many clicks are needed to go to a specific page

For example you know the article you are looking for is on page 11 but the pagination only provides pages 1-10 and a "next" link. So you have to click that "next" link, wait for the page to load, scroll down and finally select page 11. What if you wanted to go to page 100 instead?

You need to scroll down to go to the another page

Most websites provide a pagination below the content. So the user has first to scroll down to navigate to the next page. Image you would do the same with the main navigation. This is very bad practice.

What awaits you on this page? Will you find the contents you were looking for?

When you click on a pagination link, you have to wait for the new page to load and scan the page if your desired content is there.

If it isn´t there you have to go to the next page, wait, scan, etc.

So how can we fix those problems?

  1. It´s better to provide a pagination below AND above the content. So the users don´t have to scroll down the whole page, just to to go another one.
  2. Provide a scrollable pagination, so the user can directly go to the page he wants.
  3. Show a preview of the contents (for example headlines)

Some years ago the idea to use a slider-based navigation with ajax tooltips came into my mind.

So i developed this plugin in 2012 but never released it. But now its time has come to shine!

What about touch devices?

The tooltips are used best on desktop browsers as they appear on hover. Touch devices don´t support hover events. What to do about it? Show the tooltip on the first click and if one clicks again redirerect him to the page? That is not what the user expects. Don´t show anything, just go to the page? I need your help to develop a good idea for this.

Requirements

jQuery >= 1.7

but you can spice up the script with

  • animate.css - CSS3 animation for the tooltip (some more than the default i provided)
  • sly (JavaScript library for one-directional scrolling with item based navigation support)
  • jQuery-mouseWheel

Updates

I only maintain this plugin in my spare time, so if you want to make it better, please fork it on github and send me a pull-request.

Also you can write improvements in the issues tracker on github.

If you need quick fixes or custom programming you can pay me and i develop to your needs.

Possible Future Features

  • Generate pagination out of an json object, to produce less overhead. Use micro-templating eninge for that.
  • add rel="next" and rel="prev" to the links
  • Add clickable (on non touch devices) and swipeable (on touch devices) overlays to the left and right of the page, so you can easily go to the corresponding page
  • Recalculate pagination slider on window resize
  • Tooltip position should take page dimensions into account

Options:

Name Type default Description
selector String h2 Get which selectors on the target page
animation String fadeInDown classname for the CSS3 animation that gets added to the tooltip
timeout Integer 250 Timeout in ms when the tooltip should be triggered. This is to prevent unintendent triggering of the tooltips. Altough this is a very simple solution. You could use jquery hoverIntent for better handling, but support for that has to be added first
